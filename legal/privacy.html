Privacy & Security

We don't want your data. We don't have your data. Here's exactly what that means.

Overview

Strova is free, open-source animation software licensed under GPL v3. We're not a corporation trying to monetize your information. We're not a startup preparing for acquisition. We're builders who hate the modern internet's surveillance economy as much as you do.

This document explains in plain English what information flows through Strova, where it goes, and why. No legal jargon. No hidden clauses. If anything here is unclear, tell us and we'll fix it.

The one-sentence version: Strova runs entirely on your computer, collects nothing by default, and only talks to our servers when you explicitly ask it to (like checking for updates).

What We Collect

When you download Strova from www.strova.art, we see:

  • Your IP address (standard server logs, deleted after 30 days)
  • Which version you downloaded
  • Your operating system (from the user agent string)

That's it. We use this to count downloads and see which platforms need more attention. We don't sell this data. We don't feed it to machine learning models. We don't share it with partners because we don't have any.

What We Don't Collect

Let's be exhaustive here because this matters:

We don't collect your name, email, or any account information
We don't collect your animations, drawings, or project files
We don't collect your usage patterns or which features you use
We don't collect your computer's hardware specifications beyond OS type
We don't collect your location beyond what your IP address reveals
We don't use cookies, tracking pixels, or browser fingerprinting
We don't have user accounts, so we can't leak them

Local-First Design

Strova is designed to work without internet. You can animate on an airplane, in a bunker, or on a computer that never touches a network cable. Your .strova project files are plain data that lives where you put them.

This means:

  • No cloud storage that can be hacked or subpoenaed
  • No subscription required to access your own work
  • No "sync errors" that corrupt your timeline
  • No vendor lock-in—your files are yours forever

We can't delete your work because we never had it. We can't lock you out because there's no account to lock. If we disappear tomorrow, your software keeps working.

Open Source Guarantee

Strova's entire codebase is public on GitHub. This isn't just marketing—it's a privacy feature. Thousands of developers can (and do) audit every line of code. If we tried to add spyware, someone would notice immediately and we'd be publicly shamed.

You can verify our claims yourself:

  • Check the network tab—see exactly what connections Strova makes
  • Build from source—compile your own binary if you don't trust our downloads
  • Audit the code—look for any data collection in the C++ source
  • Fork the project—remove any feature you don't like

This is the difference between "trust us" and "verify yourself." We prefer the latter.

Network Activity

Strova makes exactly two types of network connections:

Update Checks

Once per day (only if enabled), Strova pings our server to see if a new version exists. This sends:

  • Your current version number
  • Your operating system
  • Your IP address (unavoidable, not stored)

You can disable this entirely in Settings → General → "Check for updates automatically."

Asset Downloads

If you browse and download brushes, palettes, or templates from our built-in asset library, that connection is obvious—you clicked it. We log what was downloaded (to see what's popular) but not who downloaded it.

That's it. No telemetry. No analytics. No "phoning home" to check your license.

Crash Reports

If Strova crashes, you'll see a dialog asking if you want to send a report. This is opt-in, not automatic. If you say yes, we receive:

  • The error message and stack trace (what function crashed)
  • Your Strova version and operating system version
  • A random crash ID (to prevent duplicate reports)

We explicitly do NOT receive:

  • Your project file or any part of it
  • Your file names or folder structure
  • Screenshots of your work
  • Your name, email, or any identifier

You can disable crash reporting in Settings → Privacy → "Send crash reports."

Third Parties

We don't use Google Analytics on our website. We don't embed Facebook pixels. We don't have "social sharing" buttons that track you. Our documentation site has no cookies.

Our website is static HTML hosted on a server we control. No Cloudflare, no AWS tracking, no third-party JavaScript libraries that could change behavior without our knowledge.

The only external service we use is GitHub for hosting our source code and issue tracker. When you visit github.com/JHPate1/Strova, you're under GitHub's privacy policy, not ours.

Your Rights

Because we don't collect personal data, GDPR "right to be forgotten" requests are technically unnecessary—we have nothing to delete. But if you want confirmation or have concerns:

  • Access: You can see everything we know about you by checking our server logs (email us)
  • Deletion: Ask and we'll delete any server logs containing your IP
  • Portability: Your project files are already portable— they're on your drive
  • Objection: Block our domain in your firewall if you want zero contact

Changes to This Policy

If we ever change this policy, we'll:

  • Post the change on our devlog with a clear explanation
  • Update the "Last updated" date below
  • Never retroactively apply changes to data already collected

If we ever add data collection (unlikely), it will be opt-in, not buried in an update.
